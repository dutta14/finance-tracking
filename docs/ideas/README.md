# Ideas Backlog

Raw product ideas and UX observations. Each file is a lightweight PM seed — enough context to generate a full 7-layer spec when promoted to an issue.

## Format

Each idea file should contain:

- **Problem** — What's broken, missing, or suboptimal
- **Evidence** — Screenshots, user friction, audit findings, or data
- **Rough direction** — Initial thinking on what the solution looks like
- **Scope signal** — Small (1 issue), Medium (2-3 issues), or Large (epic)

## Workflow

1. Drop ideas here anytime (`docs/ideas/<slug>.md`)
2. When ready to build, pick an idea and run the full spec pipeline (Architect → PM → Designer → Copywriter → Engineer specs)
3. The pipeline produces a GitHub issue with all 7 layers
4. Delete the idea file after the issue is created
